:ruby => 'História'

  • criada em 1993 no Japão
  • autor: Yukihiro Matsumoto (Matz)
  • só em 2000 foi editado o primeiro livro em inglês
  • a partir de 2005 a popularidade subiu a par do Ruby on Rails

:ruby => 'A Linguagem'

  • interpretada
  • tipologia dinâmica
  • aplicação generalizada
  • orientada a objectos
  • gestão de memória automática
  • reflexiva (meta-programming)
  • código aberto (licença FreeBSD)
  • semelhanças na syntax com Perl, Python e Java
  • influências: Perl, Smalltalk, Lisp...
Versão actual: 1.9.3

:ruby => 'aspectos da syntax'

  • iterações e blocos
    lista = ['a', 'b', 'c'] lista.each { |elemento| print elemento }
  • abc» ['a', 'b', 'c']
  • tudo tem um valor
    x = [1, 2, 3] for i in x do print i end
  • 123» [1, 2, 3]
  • tudo é um objecto
    3.times { |i| print i }
  • 012» 3

    Meta-Framework (composto por vários frameworks)

    • Active Record: abstracção da base de dados
    • Active Support: bibliotecas de ajuda ao programador
    • Action Pack: responsável pelo VC de MVC
    • Active Mailer: responsável pela automatização de mails

    :rails => 'Dependências'

    :rails => 'é um receita'

    O framework em si, e especialmente a comunidade que o suporta, incentivam à aceitação de alguns principios:

    • DRY - Don't Repeat Yourself (Ruby)
    • Convenção antes de configuração
    • Facilita a construcção de Restfull API's ao invés de SOAP-WSDL
    • Promove o conceito de Agile Development (BDD)
    • Recomenda a realização de testes (TDD)
